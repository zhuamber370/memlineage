# MemLineage v0.1.2 (Draft)

## Release Summary

`v0.1.2` is a dashboard-focused release that improves daily operator visibility and navigation speed.
It introduces a new Home Dashboard (`/`) with cross-surface snapshot metrics, deep links into task/change/knowledge workflows, and stability fixes for rapid task switching scenarios.

## Highlights

- New Home Dashboard route (`/`) with a consolidated operational view
- Three dedicated boards:
  - Task Board
  - Changes Board
  - Knowledge Board
- Query-driven deep links from dashboard cards into `/tasks`, `/changes`, and `/knowledge`
- Deterministic focus-task ranking for more predictable prioritization
- Stability hardening for task switching and execution panel data consistency
- Dashboard readability polish (chart/focus board scanability)

## Added

- Home Dashboard page and navigation entry
- Global Snapshot metrics for quick workload overview
- Unified dashboard aggregation layer in frontend to compute snapshot + focus data
- Task/Changes/Knowledge board sections with localized copy
- Deep-link filter hydration support:
  - `/tasks`: `status`, `priority`, `topic_id`, `workspace`, `task_id`
  - `/changes`: `status`
  - `/knowledge`: `status`, `category`

## Improved

- Focus task ordering is now deterministic and rule-based:
  - `P0 + in_progress` first
  - then due date
  - then latest update time
  - then title tie-break
- Dashboard cards and focus actions now open target pages with actionable filter context
- Board-level isolation for loading/error/empty states prevents full-page degradation from a single API failure
- Chart and focus-board visual polish for better daily scanning

## Fixed

- Fixed `Open Studio` task selection race condition when switching tasks quickly
- Fixed stale DAG/log replay caused by in-flight request backfill during task changes
- Hardened dashboard empty/error states
- Corrected task multi-status grouping order for more consistent list behavior
